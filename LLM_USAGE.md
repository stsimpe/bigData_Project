# Δήλωση Χρήσης Εργαλείων LLM

Σύμφωνα με τους όρους υποβολής της εξαμηνιαίας εργασίας, δηλώνω
ειλικρινά και αναλυτικά τη χρήση εργαλείων LLM ανά ενότητα της εργασίας
(κώδικας / κείμενο αναφοράς / setup). Τα εργαλεία LLM που χρησιμοποιήθηκαν
ήταν το **ChatGPT** (ανάπτυξη) και ένας **βοηθός LLM** για debugging και
έλεγχο πριν την υποβολή.

| Ενότητα | Εργαλείο | Έκταση χρήσης | Δική μου εργασία (χωρίς LLM) |
|---------|----------|----------------|------------------------------|
| **Query 1** (DataFrame, DataFrame+UDF, RDD) | — | Καμία | **100% δικό μου** — ανάλυση, επιλογή API, υλοποίηση όλων των transformations |
| **Query 2** (DataFrame, SQL) | ChatGPT / LLM | Μικρή βοήθεια στην αρχική υλοποίηση. Επιπλέον, με LLM εντοπίστηκε ότι το query έβγαζε **κενό αποτέλεσμα** στο cluster (το `DATE OCC` στο HDFS έχει μορφή `2010 Feb 20 12:00:00 AM` = `yyyy MMM dd`, διαφορετική από το sample) και διορθώθηκε το date parsing με `coalesce`/`to_date` | Η βασική λογική, το ranking ανά έτος, η εκτέλεση στο cluster, ο έλεγχος των logs και η επιβεβαίωση του διορθωμένου αποτελέσματος είναι δικά μου |
| **Query 3** (DataFrame, RDD, join hints) | ChatGPT | **Έγραψε εξ ολοκλήρου τον κώδικα** — είναι από τα σύνθετα queries (parsing GeoJSON, explode/flatMap, join census↔income, υπολογισμός per-capita) | Κατανόηση γραμμή-γραμμή, έλεγχος ορθότητας απέναντι στα δεδομένα, επιλογή και δήλωση των παραδοχών (median≈mean, διετία 2020-2021), ενσωμάτωση στο project |
| **Query 4** (DataFrame, join hints, scalability) | ChatGPT | **Έγραψε εξ ολοκλήρου τον κώδικα** — σύνθετο (Haversine ως column expressions, `crossJoin`+`broadcast`, εύρεση πλησιέστερου με `F.min(F.struct(...))`, φιλτράρισμα συντεταγμένων) | Κατανόηση και έλεγχος, σχεδιασμός των 6 executor configurations για την κλιμακωσιμότητα, ερμηνεία αποτελεσμάτων |
| **Μετατροπή CSV→Parquet** (Ζητούμενο 1) | ChatGPT | Μικρή βοήθεια (επαναχρησιμοποιεί τη λογική του Q1) | Η δομή του benchmark και η σύγκριση formats δικά μου |
| **Βοηθητικά modules** (`src/common/`: `paths.py`, `spark_session.py`, `timing.py`) | ChatGPT | Τα έφτιαξε το LLM **με δική μου καθοδήγηση** ως προς το τι θέλω (επιλογή paths local/HDFS μέσω `DATA_MODE`, σταθερές ρυθμίσεις `SparkSession`, context manager μέτρησης χρόνων) | Οι σχεδιαστικές αποφάσεις (μοτίβο `DATA_MODE`, ρυθμίσεις Spark, τρόπος μέτρησης) και η ενσωμάτωση/έλεγχος είναι δικά μου |
| **Scripts εκτέλεσης** (`scripts/*.sh`: `cluster_run_all`, `cluster_experiments`, `collect_timers`, `submit`, `run_all_configs`) | ChatGPT | Τα έφτιαξε το LLM **με δική μου καθοδήγηση** ως προς το τι θέλω να κάνουν (executor configs ανά ζητούμενο, εκτέλεση στο cluster, συλλογή χρόνων) και τα ζητούμενα του project | Ο σχεδιασμός των πειραμάτων, οι επιλογές executor configurations ανά query, η εκτέλεση και ο έλεγχος στο cluster είναι δικά μου |
| **Κείμενο αναφοράς** — ανάλυση, ερμηνείες, θεωρία (CSV vs Parquet/ORC/Avro, join strategies, scalability), συμπεράσματα | ChatGPT | **Περιορισμένη:** η δομή και το μεγαλύτερο μέρος της ανάλυσης είναι δικά μου· σε ορισμένα σημεία χρησιμοποίησα LLM για να επιβεβαιώσω ή να διατυπώσω καλύτερα κάποια συμπεράσματα/ερμηνείες | Η κύρια αλγοριθμική ανάλυση, η σύγκριση των υλοποιήσεων και η τελική κρίση είναι δικές μου. To report εχει γραφτεί από εμένα με χρήση LLM μόνο στην διατύπωση και δομή του writing. |
| **Μορφοποίηση αναφοράς** (LaTeX πίνακες, layout, listings) | ChatGPT | Βοήθεια μόνο στο formatting (πίνακες, στήλες, παρουσίαση) | Όλο το ουσιαστικό περιεχόμενο των πινάκων και του κειμένου |
| **Setup περιβάλλοντος** (kubectl, kubeconfig, `spark-defaults.conf`, `core-site.xml`, HDFS permissions) | ChatGPT **σε συνεργασία με τους επίσημους οδηγούς** του αποθετηρίου `ikons/bigdata-dsml` του μαθήματος | Troubleshooting όπου κόλλησα (π.χ. `HADOOP_USER_NAME`, permissions) | Όλη η εκτέλεση των εντολών, η σύνδεση και η επιβεβαίωση στο cluster |
| **README.md / SETUP.md / GitHub how-tos** | ChatGPT | Σύνταξη και δομή των οδηγιών | Τα ουσιαστικά βήματα, οι εκδόσεις και οι ρυθμίσεις που ισχύουν για το δικό μου setup |
